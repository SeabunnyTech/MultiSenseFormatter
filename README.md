# 數據資料檔拆分腳本使用說明

## 功能說明

這個 Python 腳本可以自動化處理地質監測數據檔案,主要功能包括:

1. **讀取數據檔案**: 從指定資料夾讀取以地名開頭的 Excel 數據檔案
2. **創建目錄結構**: 根據地名自動創建對應的資料夾
3. **拆分分頁**: 將每個 Excel 檔案的分頁拆分成獨立的檔案
4. **智能分類**: 根據儀器編號檔案,自動將拆分後的檔案放入對應的「地名/儀器類型」目錄中

## 目錄結構範例

執行後會產生如下結構:

```
output/
└── 新竹縣-尖石鄉/
    ├── GNSS地表變位/
    │   ├── GP6(H1).xlsx
    │   ├── GP7(H1).xlsx
    │   ├── GP8(H1).xlsx
    │   ├── JS077-G2(TG02).xlsx
    │   └── JS077-G3(TG03).xlsx
    ├── 地表傾斜計(雙軸)/
    │   ├── GP5(H1).xlsx
    │   ├── JS077-T1(TG01).xlsx
    │   ├── JS077-T2(TG02).xlsx
    │   ├── JS077-T3(TG03).xlsx
    │   └── T5(H6).xlsx
    └── 雨量計/
        └── JS077-R(TG02).xlsx
```

## 使用方法

### 前置準備

1. 確保已安裝 Python 3.6 或更高版本
2. 安裝必要的套件:
   ```bash
   pip install pandas openpyxl
   ```

### 設定腳本

開啟 `split_excel_by_location.py`,修改主程式中的設定區域:

```python
# ===== 設定區域 =====
# 儀器編號檔案路徑
INSTRUMENT_FILE = "你的路徑/20251031_25處MT潛勢區儀器編號.xlsx"

# 數據資料檔所在資料夾
INPUT_DIR = "你的路徑/數據資料夾"

# 輸出資料夾
OUTPUT_DIR = "你的路徑/輸出資料夾"
```

### 執行腳本

在終端機/命令提示字元中執行:

```bash
python split_excel_by_location.py
```

或

```bash
python3 split_excel_by_location.py
```

## 重要功能特性

### 1. 智能儀器類型識別

腳本會依序嘗試以下匹配方式:
- 完全匹配儀器編號
- 前綴匹配(例如 JS077-G3 會匹配 JS077-G4 的類型)
- 模糊匹配
- 統一 GPS 相關類型(將「單頻GPS位移」和「GNSS地表變位」統一為「GNSS地表變位」)

### 2. 檔案名稱處理

- 自動移除檔名中的特殊字符
- 保留原始分頁名稱作為輸出檔名

### 3. 錯誤處理

- 如果找不到對應的儀器類型,會將檔案歸類到「未分類」資料夾
- 會在執行過程中顯示警告訊息

## 注意事項

1. **檔案格式要求**:
   - 數據檔案必須是 `.xlsx` 格式
   - 檔名必須包含地名格式:「縣市-鄉鎮區-其他資訊.xlsx」
   
2. **儀器編號檔案**:
   - 必須包含每個地點的分頁
   - 每個分頁需有「儀器編號」和「儀器類型」欄位

3. **執行時間**:
   - 處理時間取決於檔案大小和數量
   - 大型檔案可能需要較長時間

## 輸出資訊

執行過程中會顯示:
- 載入的地點數量
- 找到的數據檔案數量
- 每個檔案的處理進度
- 每個分頁的儀器類型識別結果
- 儲存的檔案路徑

## 疑難排解

### 問題 1: 找不到儀器類型

**症狀**: 某些分頁被歸類到「未分類」資料夾

**解決方法**:
- 檢查儀器編號檔案中是否有對應的儀器資料
- 確認分頁名稱格式是否正確
- 查看警告訊息中嘗試過的儀器編號

### 問題 2: 無法讀取檔案

**症狀**: 出現檔案讀取錯誤

**解決方法**:
- 確認檔案路徑是否正確
- 確認檔案沒有被其他程式開啟
- 檢查檔案是否損壞

### 問題 3: 套件缺失

**症狀**: ImportError 或 ModuleNotFoundError

**解決方法**:
```bash
pip install pandas openpyxl
```
